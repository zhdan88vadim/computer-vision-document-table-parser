Этот метод хорош тем, что он **не рисует контуры друг в друге**, так как он работает с пикселями по-другому. Но он также хорош тем, что находит **внешние рамки**, а не внутренние дырки.

Я переписал ваш код с интерактивными слайдерами, используя логику из этого примера (с небольшими изменениями для OpenCV и ваших нужд).

### Важное отличие от вашего кода:
В вашем коде вы искали `RETR_TREE` (внутренние дырки). Код из примера ищет **внешние границы**. Поэтому я убрал фильтр `parent_idx != -1`. Теперь код будет рисовать рамки вокруг **всех** блоков (ячеек) на промежуточном изображении, без наложений.

Вот адаптированный код со слайдерами:

```python
import cv2
import numpy as np
import ipywidgets as widgets
from IPython.display import display, clear_output
from pathlib import Path
import random
from scipy.ndimage import morphology as scipy_morph, label

# ======================= MAIN EXTRACTION FUNCTION ==================================
def box_extraction(img_path, dilation_size, std_multiplier, min_size):
    # 1. Read in grayscale
    img = cv2.imread(img_path, 0)
    
    # === LOGIC FROM STACKOVERFLOW ===
    # 2. Inner morphological gradient.
    # scipy expects a specific footprint. We create a 3x3 matrix of 1s.
    footprint = np.ones((dilation_size, dilation_size), dtype=bool)
    
    # dilate the image and subtract the original to get the 'inner gradient' (edges)
    dilated = scipy_morph.grey_dilation(img, footprint=footprint)
    im = dilated.astype(np.int32) - img.astype(np.int32)

    # 3. Binarize using Mean + Std (Dynamic threshold)
    mean, std = im.mean(), im.std()
    t = mean + (std * std_multiplier) # We allow tuning the multiplier
    
    # Apply the threshold
    im[im < t] = 0
    im[im >= t] = 1

    # 4. Connected components (Scipy label)
    lbl, numcc = label(im)
    
    # 5. Size threshold and Box extraction
    box_coords = []
    
    # We create a blank black image to draw our final boxes on (Intermediate output)
    img_color = np.zeros_like(img)
    img_color = cv2.cvtColor(img_color, cv2.COLOR_GRAY2BGR)
    
    for i in range(1, numcc + 1):
        py, px = np.nonzero(lbl == i)
        
        # If the component is too small, ignore it
        if len(py) < min_size:
            im[lbl == i] = 0
            continue

        xmin, xmax, ymin, ymax = px.min(), px.max(), py.min(), py.max()
        
        # Calculate bounding box
        x, y, w, h = xmin, ymin, (xmax - xmin), (ymax - ymin)
        
        # Generate a random color
        color = (random.randint(100, 255), random.randint(100, 255), random.randint(100, 255))
        
        # Draw the rectangle ON THE INTERMEDIATE BLACK IMAGE
        cv2.rectangle(img_color, (x, y), (x + w, y + h), color, 2)

    # 6. DISPLAY RESULTS
    clear_output(wait=True)
    
    print(f"Всего компонент (блоков) найдено: {numcc}")
    print(f"Отфильтровано и нарисовано: {len([i for i in range(1, numcc+1) if len(np.nonzero(lbl == i)[0]) >= min_size])}")
    print(f"Параметры: Dilation={dilation_size}, Std Multiplier={std_multiplier:.2f}, MinSize={min_size}")
    
    from IPython.display import Image as IPImage
    is_success, buffer = cv2.imencode(".jpg", img_color)
    display(IPImage(data=buffer.tobytes()))

# ======================= SETUP SLIDERS ================================
Path("./output/").mkdir(parents=True, exist_ok=True)

# Слайдеры (адаптированные под параметры этого метода)
slider_dilation = widgets.IntSlider(value=3, min=1, max=9, step=2, description='Dilation Size:')
slider_std = widgets.FloatSlider(value=1.0, min=0.0, max=3.0, step=0.1, description='Std Multiplier:')
slider_min_size = widgets.IntSlider(value=200, min=50, max=5000, step=50, description='Min Size (px):')

# ==================== BIND EVENTS ====================
def update_image(change):
    box_extraction(
        "/media/vadim/1TB_SSD/my_github/computer-vision-document-table-parser/input_images/1_rotate.jpg", 
        slider_dilation.value, 
        slider_std.value, 
        slider_min_size.value
    )

slider_dilation.observe(update_image, names='value')
slider_std.observe(update_image, names='value')
slider_min_size.observe(update_image, names='value')

# ==================== ОТРИСОВКА ====================
display(widgets.VBox([slider_dilation, slider_std, slider_min_size]))

# Initial run
update_image(None)
```

### Что изменилось по сравнению с вашим кодом:
1. **Другой алгоритм:** Вместо поиска горизонтальных/вертикальных линий, мы применяем `grey_dilation` и вычитаем оригинал. Это создает **внутреннюю границу** вокруг блоков.
2. **Динамический порог:** Вместо фиксированного `128` мы используем формулу `mean + (std * multiplier)`. Слайдер `Std Multiplier` позволяет вам настраивать чувствительность детекции границ.
3. **Без иерархии (`RETR_TREE`):** Мы используем `scipy.ndimage.label`, которая просто группирует соседние пиксели. Это полностью исключает "рисование одного прямоугольника внутри другого", так как она не знает о вложенности.
4. **Рисование на черном фоне:** Как вы и просили, вывод рисуется на промежуточном черном изображении, которое создается `np.zeros_like(img)`. Теперь это выглядит как чистые цветные коробки без лишнего шума.

### Как настраивать слайдеры (новые параметры):
*   **`Dilation Size` (Размер ядра):** 
    *   Если линии тонкие и блоки не замыкаются — увеличивайте (5 или 7).
    *   Если всё слипается в одну сплошную кучу — уменьшайте до 1 или 3.
*   **`Std Multiplier` (Порог четкости):**
    *   Если вы видите слишком много мелкого мусора — увеличивайте до `1.5` или `2.0`. Это делает порог выше.
    *   Если блоки пропадают — уменьшайте до `0.5`.
*   **`Min Size` (Минимальный размер):**
    *   Увеличивайте это значение, чтобы убрать самые маленькие квадратики и оставить только большие прямоугольные ячейки.
