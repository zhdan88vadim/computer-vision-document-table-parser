# Подробное объяснение каждого этапа обработки изображения

## 🎯 Общая цель алгоритма
Алгоритм ищет **прямоугольные объекты** (например, таблицы, ячейки, блоки текста) на изображении. Он использует **морфологический градиент** для выделения границ и **анализ связных компонент** для группировки пикселей в объекты.

---

## 📍 Этап 1: Исходное изображение (градации серого)

**Что происходит:**
- Цветное изображение преобразуется в черно-белое (оттенки серого)
- Каждый пиксель получает значение от 0 (черный) до 255 (белый)
- Это необходимо, так как большинство алгоритмов обработки работают с одноканальными изображениями

**Зачем:**
- Упрощает обработку (один канал вместо трех)
- Уменьшает вычислительную нагрузку
- Большинство алгоритмов компьютерного зрения работают с яркостью, а не с цветом

**Пример:**
```
Цветное: [R=120, G=100, B=80] → Серое: 100
Цветное: [R=255, G=255, B=255] → Серое: 255
Цветное: [R=0, G=0, B=0] → Серое: 0
```

---

## 📍 Этап 2: Дилатация (расширение светлых областей)

**Что происходит:**
- Для каждого пикселя берется **максимальное** значение в его окрестности (kernel_size × kernel_size)
- Светлые области **расширяются**, темные - **сжимаются**
- Размер окна задается параметром `dilation_size`

**Зачем:**
- **Увеличивает** яркие объекты (границы, линии)
- **Заполняет** маленькие дыры и разрывы в линиях
- **Соединяет** близкие объекты в один

**Как работает (пример с окном 3×3):**
```
Исходные значения:          После дилатации:
[10, 20, 30]                [30, 30, 50]
[40, 50, 60]    →           [50, 60, 70]
[70, 80, 90]                [80, 90, 90]

Темные пиксели (10,20) стали светлее, так как рядом есть светлые (90)
```

**Визуализация эффекта:**
```
До дилатации:     После дилатации:
░░░░░             █████
░░░█░             ████░
░░█░░    →        ░███░
░█░░░             ░███░
░░░░░             ░░██░
```

## 📍 Этап 3: Морфологический градиент (ключевой этап!)

**Что происходит:**
- Из дилатированного изображения **вычитается** исходное
- Формула: `Градиент = Дилатация - Оригинал`

**Зачем:**
- Выделяет **границы объектов**
- На границах разница между дилатацией и оригиналом максимальная
- Внутри однородных областей разница минимальная

**Как работает математически:**
```
Пиксель на границе объекта:
- Исходное значение: 100 (серый)
- Дилатация: 255 (белый, так как рядом есть яркий пиксель)
- Градиент: 255 - 100 = 155 → ЯРКИЙ (граница!)

Пиксель внутри объекта:
- Исходное значение: 200 (светлый)
- Дилатация: 210 (немного увеличился)
- Градиент: 210 - 200 = 10 → ТЕМНЫЙ (не граница)

Пиксель на фоне:
- Исходное значение: 30 (темный)
- Дилатация: 35 (немного увеличился)
- Градиент: 35 - 30 = 5 → ТЕМНЫЙ (не граница)
```

**Почему это работает:**
1. На **границах** объектов происходит резкий перепад яркости
2. Дилатация "заливает" светлые области за границу
3. Разница показывает, насколько сильно изменился пиксель
4. Чем сильнее изменение, тем вероятнее это граница

**Визуальный пример:**
```
Исходное изображение:    Дилатация:          Градиент (результат):
███████████              ████████████        ░░░░░░░░░░░░
█░░░░░░░░░█              ████████████        ░██████████░
█░░░░░░░░░█    →         ████████████   →    ░██████████░
█░░░░░░░░░█              ████████████        ░██████████░
███████████              ████████████        ░░░░░░░░░░░░

Градиент показывает только границы (яркие линии)!
```

## 📍 Этап 4: Бинаризация (превращение в черно-белое)

**Что происходит:**
- Вычисляется **порог** по формуле: `Порог = Среднее + Стандартное_отклонение × Множитель`
- Все пиксели с яркостью **выше** порога становятся **белыми** (1)
- Все пиксели с яркостью **ниже** порога становятся **черными** (0)

**Зачем:**
- Превращает градиент в четкое бинарное изображение
- Отделяет **границы** от **фона**
- Упрощает поиск объектов

**Почему порог динамический (адаптивный):**
```
Темное изображение:     Светлое изображение:
Среднее = 30           Среднее = 150
Станд.откл. = 20       Станд.откл. = 25
Порог = 30 + 20 = 50   Порог = 150 + 25 = 175

При фиксированном пороге 100:
- Темное: все пиксели < 100 → все черное (провал!)
- Светлое: все пиксели > 100 → все белое (провал!)
```

**Что означают пиксели после бинаризации:**
- **Белые (1)**: Это границы объектов (то, что нас интересует)
- **Черные (0)**: Это фон (не интересно)

**Как работает порог (пример):**
```
Градиент (значения 0-255):
[120, 5, 8, 200, 45, 150, 10, 180]

Среднее = 90, Стандартное отклонение = 75
Порог = 90 + 75 = 165

Результат бинаризации:
[0, 0, 0, 1, 0, 0, 0, 1]
Только пиксели 200 и 180 стали белыми (это границы)
```

## 📍 Этап 5: Связные компоненты (Connected Components)

**Что происходит:**
- Алгоритм находит все **связанные** группы белых пикселей
- Каждая группа получает **уникальный номер** (метку)
- Пиксели считаются связанными, если они касаются (по горизонтали, вертикали или диагонали)

**Зачем:**
- Группирует отдельные пиксели в **объекты**
- Каждая компонента = потенциально один прямоугольник
- Позволяет анализировать объекты по отдельности

**Как работает (пример):**
```
Бинарное изображение:    Метки (каждый номер = объект):
1, 1, 0, 0, 2            1, 1, 0, 0, 2
1, 0, 0, 0, 2    →       1, 0, 0, 0, 2
0, 0, 0, 3, 3            0, 0, 0, 3, 3
0, 0, 0, 3, 0            0, 0, 0, 3, 0

Найдено 3 объекта!
Объект 1: 3 пикселя (слева-вверху)
Объект 2: 3 пикселя (справа)
Объект 3: 4 пикселя (снизу-справа)
```

**Важные параметры:**
- **Связность 8**: учитывает диагональные соседи (используется по умолчанию)
- **Связность 4**: учитывает только горизонтальных и вертикальных соседей

## 📍 Этап 6: Отфильтрованные компоненты (фильтрация по размеру)

**Что происходит:**
- Для каждой компоненты вычисляется **площадь** (количество пикселей)
- Если площадь меньше `min_size` → компонента **удаляется**
- Остаются только **большие** компоненты

**Зачем:**
- Удаляет **шум** (мелкие точки, артефакты)
- Удаляет **мелкие детали** (отдельные буквы, точки)
- Оставляет только **значимые объекты** (прямоугольники, блоки)

**Почему это важно:**
```
Без фильтрации:          С фильтрацией (min_size=200):
Найдено 100 объектов     Найдено 10 объектов
- 50 точек шума          - 10 реальных прямоугольников
- 30 букв                - Все объекты значимые
- 10 маленьких пятен     - Легче анализировать
- 10 реальных объектов   - Меньше ложных срабатываний
```

**Как выбирать min_size:**
- **Маленькое значение** (50-200): Находит мелкие объекты, но больше шума
- **Среднее значение** (200-500): Хорошо для большинства случаев
- **Большое значение** (500-2000): Только крупные объекты

---

## 📍 Этап 7: Финальные прямоугольники

**Что происходит:**
- Для каждой отфильтрованной компоненты вычисляется:
  - **Минимальные/максимальные** координаты X и Y
  - **Прямоугольник**, описывающий компоненту
  - **Центр** прямоугольника
- Все прямоугольники **рисуются** на изображении
- Каждый получает **уникальный цвет** и **номер**

**Зачем:**
- Визуализация результатов
- Удобный доступ к координатам каждого объекта
- Возможность дальнейшего анализа (извлечение текста, распознавание)

**Как вычисляется прямоугольник:**
```
Компонента с пикселями: (x, y)
(10, 20), (15, 20), (20, 20)
(10, 25), (15, 25), (20, 25)
(10, 30), (15, 30), (20, 30)

Минимальные значения:
x_min = 10 (самый левый пиксель)
y_min = 20 (самый верхний пиксель)
x_max = 20 (самый правый пиксель)
y_max = 30 (самый нижний пиксель)

Прямоугольник:
x = 10, y = 20
w = 20 - 10 = 10
h = 30 - 20 = 10
```

**Информация о каждом прямоугольнике:**
- **Координаты**: (x, y) - левый верхний угол
- **Размер**: (w, h) - ширина и высота
- **Площадь**: w × h
- **Центр**: (x + w/2, y + h/2)

---

## 📊 Сводка: Что делает каждый параметр

| Параметр | Что делает | Увеличение | Уменьшение |
|----------|-----------|------------|------------|
| **dilation_size** | Размер окна для дилатации | Более толстые границы, меньше разрывов | Более тонкие границы, больше разрывов |
| **std_multiplier** | Множитель для порога | Более строгий порог (меньше объектов) | Более мягкий порог (больше объектов) |
| **min_size** | Минимальная площадь объекта | Только крупные объекты | Включает мелкие объекты |

---

## 🎯 Итоговое понимание алгоритма

1. **Градиент** выделяет границы объектов (ключевой шаг)
2. **Бинаризация** превращает градиент в черно-белое изображение
3. **Связные компоненты** группируют пиксели в объекты
4. **Фильтрация** удаляет шум и мелкие детали
5. **Прямоугольники** описывают каждый найденный объект

Этот метод особенно хорош для поиска **прямоугольных блоков** на документах, таблиц, ячеек, и других структурированных элементов!
